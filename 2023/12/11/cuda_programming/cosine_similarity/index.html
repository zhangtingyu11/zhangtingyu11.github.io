cosine_similarity函数

输入参数

lidar_features: 要计算余弦相似度的第一个张量的数据, 是M×CM\times CM×C的张量的数据区域
camera_features: 要计算余弦相似度的第二个张量的数据, 是N×CN\times CN×C的张量的数据区域
cos_similarity: 余弦相似度要存放的位置, 是M×NM\times NM×N的张量的数据区域
lidar_num: 上面维度里面的MMM
camera_num: 上面维度里面的NNN
feature_dim: 上面维度里面的CCC

boxes3d_idx表示计算到了lidar_features的MMM那一维的哪一个索引
boxes2d_idx表示计算到了camera_features的NNN那一维的哪一个索引
lidar_feature表示当前要处理的lidar_features数据的位置
camera_feature表示当前要处理的camera_features数据的位置
target_pos表示当前要填写的cos_similarity的位置

因为申请block块和线程的时候会上取整, 因此会存在一些多余的线程, 所以需要判断当前的线程是不是有效线程

inner_product计算向量的内积, lidar_distance计算lidar_features在当前处理的索引上的向量的2范数, camera_distance计算camera_features在当前处理的索引上的向量的2范数

问题汇总

问题一

vscode提示如下信息:

Use of undeclared identifier “cudaConfigureCall”

解决办法:
参考这篇博客
因为我用的是CUDA 11.8, 可能也是因为没有version.txt
在/usr/local/cuda-11.8下创建version.txt并写入

问题二

vscod
提示如下信息:

GPU arch sm_20 is supported by CUDA versions between 7.0 and 8.0 (inclusive), but installation at /usr/local/cuda-11.8 is 10.1. Use --cuda-path to specify a different CUDA install, pass a different GPU arch with --cuda-gpu-arch, or pass --no-cuda-version-check.

解决办法:
在.vscode/setting.json中添加"--cuda-gpu_arch=sm_75"这一行

虽然RTX4090的计算能力是sm_89, 但是clangd插件似乎不支持, 不知道是不是版本比较低, 改低一点是好用的. 个人感觉这个仅作为静态检测, 暂时还没有出现问题
