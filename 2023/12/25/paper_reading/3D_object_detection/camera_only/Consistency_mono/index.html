Consistency of Implicit and Explicit Features Matters for Monocular 3D Object Detection(吐槽)

主要思路

正常的方法是将体素投影到图像上获得两个特征, 现在是让image backbone生成两个图像特征, 一个作为key, 一个作为value, 使用位置作为query来引入注意力机制来更新voxel特征

Introduction

单目相机3D目标检测经常会利用3D中间表示的方法: 在第一阶段获取投影后的图像特征转到3D空间, 在第二阶段使用检测头估计3D包围框

3D中间表示有以下的优点:

  • 3D空间中的物体的距离比2D空间更有代表性
  • 物体的尺度在3D空间是一致的, 与深度无关, 因此可以更好的在3D空间中检测出来
  • 3D表示可以转化成BEV表示, 送入轨迹规划的网络中

在现有的方法中, 相机的参数和预测的像素深度可以将特征从2D转化到3D点或者3D voxel.
如果将图像的RGB值转化到3D空间, 图像backbone就仅仅是作为一个深度预测器, 除了深度和RGB值, 它对3D表示没有任何帮助

如果是将原始的特征向量转化到3D, 这样每个点或者体素会有隐式特征, 但是这些特征可能与直接用3D空间中的点和体素提取出来的显式特征不一致. (例如物体的朝向转化为隐式特征可能与欧式空间观察到的朝向不一致. 因为相机的安装错误或者3D世界旋转的数据增强会导致在相机坐标系和自车坐标系下的航向角偏移较大)

主要贡献

  • 提出了orientation-aware image backbone来消除3D表示下隐式和显式航向角的差异
  • 引入local ray attention机制来将图像特征转化为voxel表示
  • 提出Gaussian位置编码, 性能上比正弦位置编码要好

Method

Overview

本文的工作包含一个orientation-aware image backbone, local ray attention, BEV backbone, 如下图所示

给定一张图片, 2D backbone会生成两个特征图, 一个是depth keys, 另一个是图像特征

高斯位置编码将隐式航向和显式航向的差异编码成一个张量, 然后将其拓展到和图像特征尺寸一致
然后将图像特征和拓展后的航向差异进行拼接生成image value

将depth key, image values和voxel的坐标送入local ray attention中生成场景的3D表示
基于3D表示, BEV backbone预测3D包围框和3D占用图(occupancy map)

Local Ray Attention

local ray attention直接将图像的特征图转化为3D体素特征, 没有中间的伪点云

local ray attention的输入是从orientation-aware backbone得到的depth key和image value以及生成的体素中心点坐标

通过相机的外参, 可以将体素的中心点从自车坐标系转换到相机坐标系中
给定相机的内参, 可以将每个体素的中心投影到图像平面获得像素坐标
通过双线性插值可以得到对应像素点的depth key和image features
体素中心在相机坐标系下的Z坐标会和高斯位置编码结合作为query

给定key, query, value, 可以更新体素的特征

v^⃗=sim(k⃗,q⃗)v⃗\vec{\hat{v}}=sim(\vec{k},\vec{q})\vec{v} v^=sim(k,q​)v
