思路

考虑几种情况, 比如sicksicksick数组为[3,7][3, 7][3,7], n=10n=10n=10
那么0,1,20, 1, 20,1,2这三个位置只能从333传过去
8,98, 98,9这两个位置只能从777这个位置传过去
而4,5,64, 5, 64,5,6这三个位置可以从3传过去, 也可以从7传过去

考虑最后的答案, 0,1,20, 1, 20,1,2这三个数在答案中的顺序一定是固定的, 是2,1,02, 1, 02,1,0的顺序
而8,98, 98,9这两个数在答案中的顺序也是固定的, 是8,98, 98,9的顺序
可以很明显的得出, 从[0,sick[0]−1][0, sick[0]-1][0,sick[0]−1]这些位置在答案中的顺序是固定的, 只有一种排列情况
从[sick[−1]+1,n−1][sick[-1]+1, n-1][sick[−1]+1,n−1]这些位置在答案中的顺序也是固定的, 也只有一种排列情况

而4,5,64, 5, 64,5,6这三个位置的顺序是很多的, 可以是[4,6,5],[6,5,4],[4,5,6],[6,4,5][4, 6, 5], [6, 5, 4], [4, 5, 6], [6, 4, 5][4,6,5],[6,5,4],[4,5,6],[6,4,5], 考虑这些排列总共有多少种
可以想象每次从两边的任意一边选择元素, 每次选择都有两种情况, 直到只剩最后一个元素, 就有2(3−1)2^{(3-1)}2(3−1)种情况

那么就可以把问题拆分成很多个区间, 每个区间都有各自的顺序, 区间和区间之间的元素顺序没有要求
假设有mmm个区间, 每个区间有ci,i=1,⋯ ,mc_i, i=1, \cdots, mci​,i=1,⋯,m个排列方式, 有cnticnt_icnti​个元素, 答案的长度为totaltotaltotal
那么问题变成了从totaltotaltotal中选择c1c_1c1​个位置, 填入任意一种排列, 然后再从total−cnt1total-cnt_1total−cnt1​个位置中选择c2c_2c2​个位置, 填入任意一种排列, 这就是一个组合数学的问题

那就涉及到2k2^k2k modmodmod ppp 和组合数取模的问题
2k2^k2k modmodmod ppp可以使用快速幂来解决, 组合数取模可以使用费马小定理
可以将组合数公式列出来

Cnm=n!m!(n−m)!,Cn0=1.C_n^m=\frac{n!}{m!\left(n-m\right)!},C_n^0=1. Cnm​=m!(n−m)!n!​,Cn0​=1.
