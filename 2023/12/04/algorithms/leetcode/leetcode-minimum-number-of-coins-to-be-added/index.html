Leetcode2952. 需要添加的硬币的最小数量

发表于2023-12-04|更新于2023-12-18|算法贪心
|字数总计:537|阅读时长:2分钟|阅读量:

题目链接

思路

看到子序列问题, 第一个想到的应该是答案和原始数组的顺序有没有关系, 如果没有关系直接排序

然后我想到的是能用前kkk个数凑出哪些值, 再考虑更一般的情况, 能用两个数凑出哪些值
比如[1,2][1, 2][1,2]能凑出来[1,2,3][1, 2, 3][1,2,3], [1,4][1, 4][1,4]能凑出来[1,4,5][1, 4, 5][1,4,5]
那么对于[1,4,5][1, 4, 5][1,4,5], 想凑出来222, 有两种方法, 一种是增加111, 让数组变成[1,1,4,5][1, 1, 4, 5][1,1,4,5]
另一种是增加222, 让数组变成[1,2,4,5][1, 2, 4, 5][1,2,4,5]

哪种更优呢? 明显增加222更优, 因为这种情况还可以凑出来333
再考虑更一般的情况, 假设前面kkk个数, 可以凑出来1,2,⋯ ,m1, 2, \cdots, m1,2,⋯,m, 但是凑不出来m+1m+1m+1, 那么想凑出来m+1m+1m+1添加哪个数更好?
答案是添加m+1m+1m+1更好, 因为这样的话1,2,⋯ ,m+m+11, 2, \cdots, m+m+11,2,⋯,m+m+1这些数都可以凑出来

那么就可以遍历排序后的coinscoinscoins数组, 然后遍历数组, 记录当前能凑出的连续序列1,2,⋯ ,m1, 2, \cdots, m1,2,⋯,m的mmm的最大值
当遍历到第iii个元素时, 如果m<coins[i]−1m < coins[i]-1m<coins[i]−1, 那么无论怎么凑都不可能凑出来m+1m+1m+1, 那么就需要增加m+1m+1m+1这个数, m=m+m+1m = m+m+1m=m+m+1, 直到m>=coins[i]−1m >= coins[i]-1m>=coins[i]−1

最后遍历完数组后如果还没达到targettargettarget, 就继续加元素m+1m+1m+1, 然后更新mmm
