FrustumFormer:Adaptive Instance-aware Resampling for Multi-view 3D Detection

发表于2023-11-30|更新于2023-12-18|论文阅读3D目标检测纯相机3D目标检测
|字数总计:2.9k|阅读时长:12分钟|阅读量:

主要思路

提出了两个query, 一个scene query(和BEV Former的query一样), 一个instance query(相当于筛选了一下感兴趣的前景query)
增加了时序信息, 是当前时间的query和过去时间的query做注意力
2023.11.30还没公开源码…可信度一般

Introduction

现有的方法中有两个方法能够将图像的特征转化为BEV视角的特征:

  1. lifting-based方法, 比如BEVDet
  2. query-based方法, 比如BEVFormer

这些方法都聚焦于视角转换策略的设计, 但是忽略了在视角转换的过程中选择正确的特征的重要性
包含车辆和行人的区域应该比天空和地面这些区域更重要, 但是之前的方法对他们一视同仁

基于上面的考虑, 提出了Adaptive Instance-aware Resampling(AIR), 主要思想是关注一部分选择的BEV queries来减少定位的不确定性

在image view上定位物体比在BEV平面上定位物体要简单. 因此在image view的proposal上面做lifting的操作, 但是这么操作得到的区域也很大. 因此提出了一个occupancy mask predictor 和 temporal frustum fusion module 来进一步减少定位的不确定性.

在BEV平面的frustum queries上学习一个occupancy mask, 用来预测一个区域是否可能包含物体
融合不同时间步的instance frustum, 区域的交集对物体的真实位置也有限制

本文提出了FrustumFormer, 在image view和bird’s eye view上建立连接, 包含两个关键设计:

  1. 一个frustum encoder通过adaptive instance-aware resampling来增强instance-aware feature
  2. 一个temporal frustum fusion module用来整合历史的instance frustum feature用来精确的定位和速度预测

主要贡献:

  • 提出FrustumFormer, 一个全新的框架, 利用了image view和BEV view之间的几何限制
  • 提出选择什么做view transformation是很有用的. 因此设计了Adaptive Instanceaware Resampling (AIR) 来更关注包含物体的区域.
  • SOTA

Method

输入是N张图像I={Ii∈R3×H×W}i=1NI=\{I_{i}\in\mathbb{R}^{3\times H\times W}\}_{i=1}^{N}I={Ii​∈R3×H×W}i=1N​ 以及相机的内参K={Ki∈R3×3}i=1NK=\{K_{i}\in\mathbb{R}^{3\times3}\}_{i=1}^{N}K={Ki​∈R3×3}i=1N​ 和相机的外参T={Ti∈R4×4}i=1NT=\{T_{i}\in\mathbb{R}^{4\times4}\}_{i=1}^{N}T={Ti​∈R4×4}i=1N​
目标是输出3D包围框

如下图所示. FrustumFormer主要关注特征的转化过程, 包含四个部分: 一个image backbone, 一个frustum encoder, 一个frustum fusion module, 一个检测头

首先使用image backbone来提取多尺度多视角特征, 然后在frustum fusion module和frustum encoder的帮助下将图像特征转化为BEV特征. 最后使用一个query-based检测头来输出检测结果

Frustum Encoder

Frustum encoder将多尺度多视角的图像特征F\mathbf{F}F转化为BEV特征B\mathbf{B}B.
使用两类BEV queries来构建最终的BEV特征: scene queries Qs\mathbf{Q}_sQs​ 和 instance queries Qi\mathbf{Q}_iQi​
Qs\mathbf{Q}_sQs​是稠密的, 通过规则的网格生成, Qi\mathbf{Q}_iQi​是稀疏的, 由不规则的instance frustum生成

scene query的学习和BEVFormer类似. instance queries通过instance region进行学习, 学习到的特征会和scene feature结合形成最终的BEV特征
instance region的选择是通过adaptive instance-aware re-sampling进行, 包含:(1) instance frustum generation (2) frustum occupancy mask prediction (3) instance frustum cross-attention

Instance Frustum Query Generation

本节介绍单个instance frustum Qf\mathbf{Q}_fQf​的生成, 是instance queries Qi\mathbf{Q}_iQi​的一个子集
主要的思想是利用perspective view的instance mask, 然后在BEV平面选择对应的区域
定义网格状的可学习的参数Qi∈RH×W×C\mathbf{Q}_i \in \R^{H \times W \times C}Qi​∈RH×W×C, 其中H,WH, WH,W是BEV queries的空间形状, CCC是通道维度

Qf={Qpi∈Qi∣∃π(pik)∈S}\mathbf{Q}_f=\{\mathbf{Q}^{p_i}\in\mathbf{Q}_i|\quad\exists\quad\pi(\mathbf{p}_i^k)\in S\} Qf​={Qpi​∈Qi​∣∃π(pik​)∈S}

其中Qpi∈R1×C\mathbf{Q}^{p_i} \in \R^{1 \times C}Qpi​∈R1×C是位于pi=(xi,yi)\mathbf{p}_i = (x_i, y_i)pi​=(xi​,yi​)位置的query, pik\mathbf{p}_i^kpik​表示位于pi\mathbf{p}_ipi​的第k个采样点. π(pik)\pi(\mathbf{p}_i^k)π(pik​)表示pik\mathbf{p}_i^kpik​的投影点
换个表述就是每个BEV的网格里面定义了K个采样点, 每个BEV网格相当于一个query, 只有这K个采样点中的其中一个点落在了这个物体的图像掩码上, 那么这个BEV 网格就是一个instance query

Frustum Occupancy Mask Prediction

尽管instance frustum提供了物体的可能的位置, 但是它在BEV平面的对应位置可能因为深度的不确定性导致很大.
接着提出一个occupancy mask
给定一个物体的所有的frustum queries ∪Qf\cup\mathbf{Q}_{f}∪Qf​(为什么这么叫, 我的理解是一个物体可能对应多个instance query, 这些query构成了一个视锥)
设计了一个OccMask模块来预测BEV平面上的一个二值的占用掩码Obev∈RH×W×1\mathbf{O}_{bev} \in \R^{H\times W \times 1}Obev​∈RH×W×1 (这个mask对所有的instacnce frustum queries是通用的)

占用掩码表示了这个网格存在物体的概率

Obev=OccMask(∪Qf)\mathbf{O}_{bev}=Occ\boldsymbol{M}ask(\cup\mathbf{Q}_f) Obev​=OccMask(∪Qf​)

其中OccMaskOccMaskOccMask是包含2D卷积的可学习模块
监督的信息来自于3D包围框在BEV平面上的投影, 使用focal loss来学习占用掩码

Lm=FocalLoss(Obev,Ω)\mathcal{L}_m=FocalLoss(\mathbf{O}_{bev},\boldsymbol{\Omega}) Lm​=FocalLoss(Obev​,Ω)

使用BEV平面的最小投影包围框作为监督的信号

在每一层, 使用上一层输出的instance frustum query来预测当前的占用掩码. 每一个frustum encoder都要预测占用掩码

Instance Frustum Cross-Attention(IFCA)用来进行instance queriesQi\mathbf{Q}_iQi​和图像特征F\mathbf{F}F的交互
instance queriesQi\mathbf{Q}_iQi​通过下面式子进行选择

Qi={Qpi∈∪Qf∣Obev(i)=1}\mathbf{Q}_i=\{\mathbf{Q}^{p_i}\in\cup\mathbf{Q}_f|\mathbf{O}_{bev}(i)=1\} Qi​={Qpi​∈∪Qf​∣Obev​(i)=1}

IFCA可以表示为下面的式子

IFCA(Qipi,Fj)=1∣v∣∑j∈v∑m=1MDA(Qipi,πj(pim),Fj)IFCA(\mathbf{Q}_i^{p_i},\mathbf{F}_j)=\frac{1}{|v|}\sum_{j\in v}\sum_{m=1}^{M}DA(\mathbf{Q}_i^{p_i},\pi_j(\mathbf{p}_i^m),\mathbf{F}_j) IFCA(Qipi​​,Fj​)=∣v∣1​j∈v∑​m=1∑M​DA(Qipi​​,πj​(pim​),Fj​)

其中Qipi\mathbf{Q}_i^{p_i}Qipi​​是位于pip_ipi​的instance query. πj(pim)\pi_j(p_i^m)πj​(pim​)是第mmm个参考点到第jjj个相机视角的投影点. MMM是instance query的采样点数. Fj\mathbf{F}_jFj​是第jjj个相机视角的图像特征. DADADA表示deformable attention. vvv是2D参考点可以落到的视角的集合, ∣v∣{|v|}∣v∣是视角的数量
思想是BEV平面的instance query作为query, 图像平面的特征作为key和value

Temporal Frustum Cross-Attention

TFCA整合历史的instance frustum queries Hf\mathbf{H}_fHf​到当前的instance frustum queries Qf\mathbf{Q}_fQf​
因为物体可能会移动, 所以使用deformable attention来减少物体移动的影响

TFCA(Qfpi,Hf)=∑m=1MDA(Qfpi,pi′m,Hf)TFCA(\mathbf{Q}_{f}^{p_{i}},\mathbf{H}_{f})=\sum_{m=1}^{M}DA(\mathbf{Q}_{f}^{p_{i}},\mathbf{p}_{i}^{'m},\mathbf{H}_{f}) TFCA(Qfpi​​,Hf​)=m=1∑M​DA(Qfpi​​,pi′m​,Hf​)

Implementation Details

使用两种backbone: ResNet101-DCN和VoVnet-99
frustum encoder包括6层
BEV queries的尺寸是200×200200 \times 200200×200
检测范围在XY轴是[−51.2m,51.2m][-51.2m, 51.2m][−51.2m,51.2m], 在Z轴是[−3m,5m][-3m, 5m][−3m,5m]
每个BEV网格采样K=8K=8K=8个点
对于BEV queries采用可学习的位置编码

对于2D instance proposal, 采用在nuImages中预训练的Mask-RCNN. 使用输出的包围框作为object mask region
感觉应该用实例分割效果好点, 目标检测里面还是有大量背景像素

占用网格的loss的权重为5

frustum fusion module使用一个长度为8的时序窗口, 从中随机采样4个关键帧
