Is Pseudo-Lidar needed for Monocular 3D Object detection?

发表于2023-11-29|更新于2023-12-18|论文阅读3D目标检测纯相机3D目标检测
|字数总计:2.4k|阅读时长:9分钟|阅读量:

主要思路

主要是和伪LiDAR的方法做对比, 移除了伪LiDAR方法中的深度finetune的过程
主要的操作就是用深度的loss进行预训练, 用其他的loss进行finetune
可以拓展成用一个loss进行预训练, 用其他的loss进行finetune

Introduction

伪LiDAR的检测方法使用一个预训练的深度网络来构建点云的表现形式. 这种伪LiDAR方法的好处是随着深度估计的精度的提高, 模型的效果也能更好

但是从单个图片中获取深度是一个病态的问题, 这也是伪LiDAR和基于LiDAR的方法差距悬殊的原因

基于伪LiDAR的方法是在一个预训练的深度网络上来优化深度网络, 最后训练预测3D框
DD3D的方法利用深度信息来预训练DD3D, 然后用3D包围框信息来优化DD3D
两者的差异就是伪LiDAR的方法需要额外的深度fine-tuning

Dense depth pre-training for 3D detection

Architecture


DD3D是在FCOS上面拓展, 进行3D目标检测和深度预测
包含一个backbone和三个子网络(检测头), 这三个检测头在多尺度的特征上面共享
backbone输入是图像, 输出的结果是多尺度的特征(按照上图, 总共有5个尺度的特征)

每个特征图都会连接三个检测头:

  1. 分类检测头预测类别
  2. 2D包围框检测头预测这个位置到2D包围框的四个边的距离(参考FCOS)
  3. 3D检测头比较复杂, 总共预测12个值:

(1) q=(qw,qx,qy,qz)\mathbf{q}=(q_w,q_x,q_y,q_z)q=(qw​,qx​,qy​,qz​)预测的是车辆的朝向与观测方向相关(allocentric)的夹角的四元数. 与相机坐标系相关的的角度称为egocentric. 注意这个四元数预测的是3个自由度的角度都要预测.
(2) z{c,p}z_{\{c, p\}}z{c,p}​. zcz_czc​表示3D包围框中心的zzz值, 只和前景的特征有关. zpz_pzp​是相机距离最近的表面的深度, 这个与全部像素有关. 我的理解是, 这个像素如果表示地面, 那就要求到地面的深度. 因为深度的范围是比较大的, 因此希望他们在同一个区间, 网络的输出并不是深度值, 需要进行如下转换

d=cp⋅(σl⋅z+μl),p=1fx2+1fy2,\begin{aligned} &d=\frac{c}{p}\cdot\left(\sigma_{l}\cdot z+\mu_{l}\right), \\ &p=\sqrt{\frac{1}{f_{x}^{2}}+\frac{1}{f_{y}^{2}}}, \end{aligned} ​d=pc​⋅(σl​⋅z+μl​),p=fx2​1​+fy2​1​​,​

其中z∈{zc,zp}z\in \{z_c, z_p\}z∈{zc​,zp​}是网络的输出, d∈{dc,dp}d \in \{d_c, d_p\}d∈{dc​,dp​}是预测的深度, (σl,μl)(\sigma_l, \mu_l)(σl​,μl​)是每个FPN层级定义的可学习的缩放和offset因子, ppp是通过焦距计算出来的像素大小

这种方法让深度的预测不仅来自于输入图片, 而且来自于像素大小, 是camera-awareness的

在训练过程中如果对图片进行了缩放, 需要修改内参矩阵

K=[rxry1][fx0px0fypy001]\mathbf{K}=\begin{bmatrix}r_x&r_y&1\end{bmatrix}\begin{bmatrix}f_x&0&p_x\\0&f_y&p_y\\0&0&1\end{bmatrix} K=[rx​​ry​​1​]⎣⎢⎡​fx​00​0fy​0​px​py​1​⎦⎥⎤​

其中rx,ryr_x, r_yrx​,ry​是缩放比例

因为特征图的大小是比原图要小的, 最后会使用双线性插值还原到原图上

(3) o=(Δu,Δv)\mathbf{o}=(\Delta_u,\Delta_v)o=(Δu​,Δv​) 表示特征的位置到3D包围框投影的位置的offset
可以通过以下公式来还原3D中心点

C=K−1[ub+αlΔuvb+αlΔv1]dc\mathbf{C}=\mathbf{K}^{-1}\begin{bmatrix}u_b+\alpha_l\Delta_u\\v_b+\alpha_l\Delta_v\\1\end{bmatrix}d_c C=K−1⎣⎢⎡​ub​+αl​Δu​vb​+αl​Δv​1​⎦⎥⎤​dc​

其中(ub,vb)(u_b, v_b)(ub​,vb​)是特征的位置, αl\alpha_lαl​是每一层FPN定义的可学习的缩放因子

(4) δ=(δW,δH,δH)\delta=(\delta_{W},\delta_{H},\delta_{H})δ=(δW​,δH​,δH​)表示和特定类别的设定尺寸的偏差, 预测的尺寸为s=(W0eδw,H0eδH,D0eδD)\mathbf{s}=(W_{0}e^{\delta_{\mathsf{w}}},H_{0}e^{\delta_{\mathsf{H}}},D_{0}e^{\delta_{\mathsf{D}}})s=(W0​eδw​,H0​eδH​,D0​eδD​), 其中(W0,H0,D0)(W_0, H_0, D_0)(W0​,H0​,D0​)是每个类别设定的尺寸

(5) β3D\beta_{3D}β3D​表示3D包围框的置信度, 用sigmoid将它转换成概率, 再将它和分类检测头输出的分数相乘作为最终的分数

Losses

2D包围框选用FCOS的损失函数

L2D=Lreg+Lcls+Lctr\mathcal{L}_{2\mathsf{D}}=\mathcal{L}_{reg}+\mathcal{L}_{cls}+\mathcal{L}_{ctr} L2D​=Lreg​+Lcls​+Lctr​

对于3D目标检测, 使用分离的L1 loss

L3D(B∗,B^)=18∣∣B∗−B^∣∣1\mathcal{L}_{\mathrm{3D}}(\mathbf{B}^{*},\hat{\mathbf{B}})=\frac{1}{8}||\mathbf{B}^{*}-\hat{\mathbf{B}}||_{1} L3D​(B∗,B^)=81​∣∣B∗−B^∣∣1​

其中B∗B^{*}B∗和B^\hat{B}B^是gt和预测包围框的8个顶点
这个loss会重复四次, 每次会将航向, 投影的中心点, 深度, 尺寸中的其中一个不变, 另外三个变成真实值

对于3D置信度, 使用自监督的loss,

p3D∗=e−1TL3D(B∗,B^)p_{3D}^{*}=e^{-\frac{1}{T}\mathcal{L}_{3D}\left(\mathbf{B}^{*},\hat{\mathbf{B}}\right)} p3D∗​=e−T1​L3D​(B∗,B^)

其中TTT是温度系数, LconfL_{conf}Lconf​是p3D和p3D∗p_{3D}\text{和}p_{3D}^{*}p3D​和p3D∗​的binary cross entropy loss
这里我理解是用回归的loss来定义分类的真实值

最终的loss为

LDD=L2D+L3D+Lconf\mathcal{L}_{\mathsf{DD}}=\mathcal{L}_{\mathsf{2D}}+\mathcal{L}_{\mathsf{3D}}+\mathcal{L}_{conf} LDD​=L2D​+L3D​+Lconf​

Depth pre-training

对于每个FPN层级, 将稀疏的点云投影到图像上, 获得像素的深度的真实值, 然后计算所有的损失和

Ldepthl=∣∣D∗−D^l∣∣1⊙M,Ldepth=∑lLdepthl,\begin{aligned}\mathcal{L}_{depth}^l&=||\mathbf{D}^*-\hat{\mathbf{D}}_l||_1\odot\mathbf{M},\\\mathcal{L}_{depth}&=\sum_l\mathcal{L}_{depth}^l,\end{aligned} Ldepthl​Ldepth​​=∣∣D∗−D^l​∣∣1​⊙M,=l∑​Ldepthl​,​

其中D∗\mathbf{D}^{*}D∗是gt, D^l\hat{\mathbf{D}}_lD^l​是第lll层FPN预测的深度图. M\mathbf{M}M表示这个像素点是不是有效的像素点(有可能有的像素点没有点云能投影上去)

思想是先利用深度的loss进行预训练, 然后再利用其他loss进行fine-tuning

Pseudo-Lidar 3D detection

伪LiDAR方法记为PL. PL是一个两阶段的方法:

  1. 首先给定一张图片, 对他使用深度网络预测每个像素的深度.
  2. 然后将深度图转化为3D点云, 应用基于LiDAR的3D目标检测器

Monocular depth estimation

单目深度预测的目的是对每个像素p∈Ip \in Ip∈I计算深度D^=fD(I(p))\hat{D}=f_{D}\left(I\left(p\right)\right)D^=fD​(I(p))
类似于上述公式, 计算深度预测的loss, 这里的loss选择SILog loss

Network architecture

选择PackNet

3D detection

首先将像素转化为3D点云, 然后将3D点云和像素值拼接, 这样会得到一个6D的张量
使用一个2D目标检测器获取感兴趣的区域, 对这个感兴趣区域里面的点应用3D检测网络生成3D包围框

Backbone, detection head and 3D confidence

backbone使用ResNet-18
根据ROI depth map计算前景的mask, 然后过滤背景点
根据距离分成三种难度, 对每个难度生成一个包围框, 根据ROI的平均深度选择使用哪个包围框作为最终的结果
对每个detection预测一个3D置信度γ\gammaγ

Loss Function

3D回归的loss为

L3DPL=Lcenter+Lsize+Lheading+Lcorners\mathcal{L}_{3D}^{\mathsf{PL}}=\mathcal{L}_{center}+\mathcal{L}_{size}+\mathcal{L}_{heading}+\mathcal{L}_{corners} L3DPL​=Lcenter​+Lsize​+Lheading​+Lcorners​

3D置信度的目标值为γ^=e−Lcorners\hat{\gamma}=e^{-\mathcal{L}_{corners}}γ^​=e−Lcorners​
最终的loss为LPL=L3DPL+LconfPL{\mathcal{L}_{\mathsf{PL}}}=\mathcal{L}_{\mathsf{3D}}^{\mathsf{PL}}+\mathcal{L}_{conf}^{\mathsf{PL}}LPL​=L3DPL​+LconfPL​

Implementation detail

DD3D

使用V2-99 + FPN作为backbone
预训练的时候先将DD3D的backbone参数初始化为用COCO数据集预训练的参数, 然后使用DDAD15M数据集预训练深度预测
在测试的时候使用了resizing和flipping的数据增强. 在KITTI上有提升, 但是在nuScenes上没有

PL.

只使用DDAD15M的前向相机来预训练PackNet, 然后用KITTI-Depth进行fine-tune
