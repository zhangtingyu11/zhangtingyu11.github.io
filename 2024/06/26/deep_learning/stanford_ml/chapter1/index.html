斯坦福2021秋季·实用机器学习(第一章)

课程介绍

常见的机器学习应用

  • 制造业: 维修预测, 质量控制
  • 零售业: 商品推荐, 聊天机器人, 需求预测
  • 医疗: 通过病人信息进行预警, 疾病诊断
  • 金融: 欺诈识别, 贷款申请的处理
  • 汽车: 故障检测, 自动驾驶

机器学习的一般流程

  • Step1: 将问题定义成一个机器学习问题
  • Step2: 收集和处理数据
  • Step3: 训练并且微调模型
  • Step4: 部署模型
  • Step5: 监控模型, 根据模型的表现, 可能需要手机更多的数据, 训练新的模型等等, 跳到Step2

在机器学习各个阶段的一些挑战
定义问题: 一般是聚焦最有影响力的工业问题
数据获取: 高质量的数据是稀少的, 并且涉及隐私问题
训练模型: 机器学习模型越来越复杂, 很吃数据, 并且很昂贵
部署模型: 巨大的计算两不适合实时预测
监控模型: 可能会发现模型存在数据偏移(训练数据和实际数据存在分布偏移)以及公平性的问题(可能数据中就带有种族歧视)

机器学习中的不同的角色

  • 领域专家: 有商业眼光, 知道什么数据是重要的, 以及在哪找到这些数据, 能够识别一个机器学习模型的影响力
  • 数据科学家: 能够进行数据挖掘, 模型训练和模型部署
  • 机器学习专家: 可以自己设计一个SOTA的机器学习模型
  • SDE: 设计和维护数据处理, 模型训练以及提供服务的通道

数据获取

常见的机器学习数据集

  • MNIST: 手写数字数据集
  • ImageNet: 百万级别的从搜索引擎上爬取的图像数据集
  • AudioSet: YouTube上面的声音切片用于声音分类
  • LibriSpeech: 从音频书上获取的1000小时的英文演讲数据
  • Kinetics: YouTube上面的视频切片用于人物动作分类
  • KITTI: 自动驾驶数据集
  • Amazon Review: 亚马逊网购的顾客评论
  • SQuADL 维基上面的问题答案对

如何找到数据集

  • Paperswithcodes数据集
  • Kaggle数据集
  • 在Google上搜索数据集
  • tensorflow, huggingface一些工具包里面也有数据集
  • 一些会议或者公司的机器学习竞赛
  • AWS上面的公开数据
  • 自己公司里面的数据集

数据整合

比如有两个数据库的表, 怎么将两个数据库的表整合起来
Inner Join就是将两个表中的交集合并起来
Left Join就是将第一个表中的数据添加上第二个表中的数据, 如果第二个表中没有第一个表中的数据, 就填空白

数据生成

有一些数据生成的方案

  • 使用GAN来进行生成
  • 使用仿真软件进行生成
  • 使用一些图像的数据增强, 或者是Back Translation, 比如将英文翻译成法语, 然后再将法语翻译成英文

网页数据抓取

网页抓取

目标是从网页上面提取数据

  • 这些数据是包含噪声的, 是弱标签, 可能是垃圾数据
  • 在规模上是很可观的

一些机器学习数据集就是通过网页爬取来获得的

  • 例如ImageNet和Kinetics

网页爬取和网页抓取

  • 网页爬取(Crawling): 就是将整个网页抓取下来, 后面如果要查找信息就看这个网页是否关联, 相当于搜索引擎
  • 网页抓取(Scraping): 获取网站网页中的一些特定的数据

网页抓取的工具

  • 有的使用"curl"没有用, 因为一些网站的拥有者会使用各种不同的方式来组织机器人抓取网页
  • 这个时候要使用没有头的浏览器, 也就是没有GUI的浏览器
  • 需要许多新的IP地址, 因为可能爬取的时候会导致IP地址被封禁, 这个可以通过公有云实现

开销

抓取100万个房屋数据的开销大概是16.6美元
每个网页需要三秒钟
使用100个云服务器需要8.3小时
可能在抓取的过程中IP被封禁了, 需要重启实例

合法性

网页抓取本身并不是不合法的
但是不应该抓取敏感数据
不应该抓取有版权的数据
不应该抓取明令禁止的数据
如果用于盈利需要咨询律师

数据标注

半监督学习

主要适用的场景是有一小部分有标注的数据和大量未标注的数据
半监督学习做了以下假设

  • 连续性假设: 相似特征的样本更容易有相同的标签
  • 聚类假设: 数据有内在的聚类结构, 相同聚类上的样本倾向于有相同的标签
  • 流型假设, 数据分布在一个低维的流型数据空间上

自学习

自学习就是先在有标注的数据集上训练一个模型, 然后预测无标签的数据, 将置信度很高的预测加入有标签的数据中

可以使用比较贵的模型来保证数据预测的准确度

众包

找很多人来进行标注, 一个分类问题的比起哦哎你是0.012刀, 标注一个包围框是0.024刀, 语义分割的一张图片是0.84刀

挑战

  • 需要简化用户交互, 比如设计简单的任务, 明确需要, 设计易于使用的界面
  • 需要的成本很高
  • 不同的标注者标注的质量都不同

设计用户交互
比如有一个数据集, 需要预测它是365个类别中的哪一类, 这个是比较难的, 但是可以判断一章图片是否是xx,这就是一个二分类问题

减少任务量
有些数据是比较容易标注的, 这些数据就不需要人来进行标注, 这种叫主动学习

  • 不确定性采样: 将最不确定的样本交给标注员, 比如这个样本对于每个类别的概率都是1/n, 其中n是类别数
  • 使用多个模型来整合结果, 选择最不确定的样本

下面的方法将自学习和主动学习结合起来

质量控制

标注员可能会犯错误
最简单但是最昂贵的办法就是将同一个任务发给多个标注员, 通过少数服从多数的方式决定标签
也可以就发给两个人, 如果两个人相同, 就不用发给更多人, 否则发给更多人
或者将一些有标注的数据给标注员标注, 如果他们标注质量很差就不使用他们

弱监督

半自动地生成标号: 比人标注的要差一些, 但是足够训练

数据编程
用启发式的程序来赋予标签
比如说设计规则来判断YouTube的评论是恶意刷的还是人评论的

比如如果评论里有check out, 可能是让你看看其他网页, 或者看看广告, 这种就是恶意刷的, 如果没有的话就认定为中性
如果用一个情绪分析模型分析这个句子, 能达到90%以上, 就是一个正面情绪, 就认为这是一个正面的评论, 否则就是中性的
人类正常的评论可能会比较短, 短的认为是人类评论的

还要一些其他方法就是用别的已有的模型来做标注
