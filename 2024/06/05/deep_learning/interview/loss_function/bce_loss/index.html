深度学习常用损失函数(BCE)

二元交叉熵损失函数(BCE)

公式

二元交叉熵损失函数的公式为

L=−1N∑i=1N[yilog⁡(y^i)+(1−yi)log⁡(1−y^i)]L=-\frac1N\sum_{i=1}^N[y_i\log(\hat{y}_i)+(1-y_i)\log(1-\hat{y}_i)] L=−N1​i=1∑N​[yi​log(y^​i​)+(1−yi​)log(1−y^​i​)]

其中NNN代表样本数量, yiy_iyi​表示第iii个样本的真实标签, yi^\hat{y_i}yi​^​表示第iii个样本的预测值

Pytorch接口

输入参数
weight是对batch中的每个样本赋一个权重, 如果指定的话, 尺寸应该和输入大小一致
size_average: 弃用, 请使用reduction
reduce: 弃用, 请使用reduction
reduction: 可以选择三个. ‘none’, ‘mean’, ‘sum’. 如果是’none’, 就不会进行reduction, 如果是’mean’, 就会除输出元素的个数, 如果是’sum’, 就会将输出元素相加

使用示例

需要注意的一点是, 输入和标签的范围都必须在0~1之间, 否则会报错

指定weight

注意weight的尺寸和输入尺寸是一致的

指定reduction为’none’

这种情况下loss的尺寸为[8, 12]

指定reduction为’mean’

这样求出来的loss会是一个标量, 会对8*12个元素求平均

指定reduction为’sum’

这样求出来的loss会是一个标量, 会对8*12个元素求和

应用场景

二分类任务, 比如判断图片中是否包含猫
当然也可以处理多标签的二分类问题, 比如总共有64个类, 类中有重叠, 那么标签的尺寸就应该是(batch_size, 64), 表示64个类, 每个类的标签, 相当于对64个类, 每个类别都做一次二分类

BCEWithLogitsLoss

介绍

这个和BCELoss的区别就是, 损失函数内部自带了sigmoid函数, 官方说这么做会提高数值稳定性

nn.BCEWithLogitsLoss()=−1N∑i=1N[yilog⁡σ(zi)+(1−yi)log⁡(1−σ(zi))]nn.BCEWithLogitsLoss()=-\frac1N\sum_{i=1}^N[y_i\log\sigma(z_i)+(1-y_i)\log(1-\sigma(z_i))] nn.BCEWithLogitsLoss()=−N1​i=1∑N​[yi​logσ(zi​)+(1−yi​)log(1−σ(zi​))]

函数原型

主要的参数和BCELoss一致, 不做过多说明, 主要说下pos_weight这个参数, 针对不同的类别, 每个类别的正样本和负样本个数可能不一样, 这样会导致正负样本不均衡, 可以使用pos_weight指定每个类别的权重, 比如如果一个类别的正样本个数是100, 负样本个数是300, 可以将权重设置为300/100=3. pos_weight的尺寸为类别的个数
