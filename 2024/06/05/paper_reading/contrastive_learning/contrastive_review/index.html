InstDisc: Unsupervised feature learning via non-parametric instance discrimination

论文地址

论文动机


作者观察到将一张豹子图片送入一个有监督学习的分类器中, 识别出来的比较靠前的类别都是和豹子相关的, 靠后的都是和豹子不相关的
作者认为这并不是因为语义标签的类似导致的, 而是因为这些图片的特征本身就很相似
作者在这种观察的基础上, 提出了个体判别这个任务, 将每个物体都视为一个类别, 目标是学习一种特征, 把每张图片都区分开

方法


作者希望使用卷积神经网络, 每张图片得到一个特征, 每个图像的特征在最后的特征空间里尽可能的分开
使用对比学习来解决这个问题, 正样本是经过数据增强后的图片, 负样本是其他图片
将所有图片的特征存在memory bank里, ImageNet里面有128w张图片, 就有128w个特征
batch_size的大小是256, 也就有256个正样本, 负样本从memory bank中抽取4096个, 更新之后的卷积神经网络再更新抽取的负样本的特征送回memory bank中

一些Trick

Proximal Regularization

给正样本的loss增加了一个正则项λ∥vi(t)−vi(t−1)∥22.\lambda\|\mathbf{v}_i^{(t)}-\mathbf{v}_i^{(t-1)}\|_2^2.λ∥vi(t)​−vi(t−1)​∥22​., 这样可以保证更新前后的特征差别不是太大, 类似于MoCo的想法, 希望编码器变化的足够小, 使特征保持一致性

−log⁡h(i,vi(t−1))+λ∥vi(t)−vi(t−1)∥22.-\log h(i,\mathbf{v}_i^{(t-1)})+\lambda\|\mathbf{v}_i^{(t)}-\mathbf{v}_i^{(t-1)}\|_2^2. −logh(i,vi(t−1)​)+λ∥vi(t)​−vi(t−1)​∥22​.

超参数设定

计算loss的时候, 温度的设置是0.07
选择了4096个负样本
训练了200个epoch
batch size是256
初始学习率是0.03

InvaSpread: Unsupervised Embedding Learning via Invariant and Spreading Instance Feature

论文地址

论文动机

没有使用额外的数据结构去存储大量的负样本, 它的正负样本来自于同一个mini-batch, 只使用同一个编码器进行端到端的学习
动机还是类似的图片出来的特征是类似的, 不相似的图片出来的特征不类似

方法

还是选择个体判别作为代理任务

batch_size设置的是256
正样本就是原始图片以及经过数据增强后的图片
正样本对的数量是256, 负样本对的数量是(256-1)*2*256, 也就是每个图片去掉它本身以及数据增强厚的图片都和当前的图片构成负样本对
这就是MoCo论文里面的端到端的学习方式, 只使用一个编码器

CPC: Representation Learning with Contrastive Predictive Coding

论文介绍

个体判别这个代理任务显然是属于判别式模型
本文使用的就是生成式的代理任务
模型不仅能处理音频, 还能处理图片文字和强化学习

xt−3x_{t-3}xt−3​, xt−2x_{t-2}xt−2​, xt−1x_{t-1}xt−1​, xtx_{t}xt​都是之前时刻的输入, 他们经过编码器gencg_{enc}genc​之后会得到特征, 然后使用一个自回归模型garg_{ar}gar​来得到当前时刻的输出ctc_tct​, 根据ctc_tct​, 可以对未来时刻的特征进行预测
对比学习如何使用?
未来时刻的语音输入经过编码器得到的特征和预测的未来时刻的特征就是正样本对
负样本对非常的广泛, 比如随便一个输入送入编码器就和未来时刻的预测是负样本对

CMC: Contrastive Multiview Coding

论文介绍

一个物体的不同视角都是一个正样本

选择了NYU RGBD这个数据集, 这个数据集有四个视角, 第一个是原始图像, 第二个是深度信息, 第三个是surface normal, 第四个是物体分割的图像
这四个图片应该互为正样本, 再随机挑一张图片和这四张图片就是负样本
每个图片用不同的编码器

MoCo:Momentum Contrast for Unsupervised Visual Representation Learning

和其他工作的区别和联系

将之前的工作归纳总结成一个字典查询的问题
提出了两个东西: 队列和动量编码器
MoCo的大部分参数和模型都是参考的InstDisc

SimCLR: A Simple Framework for Contrastive Learning of Visual Representations

论文介绍


如果有一个一个mini-batch, mini-batch里面的图片xxx经过数据增强得到x~i\tilde{x}_ix~i​和x~j\tilde{x}_jx~j​, 这两张图片就是正样本, 如果batch size是NNN, 那么正样本对的个数就是NNN, 负样本对的个数就是2×N×(N−1)2\times N \times (N-1)2×N×(N−1)
x~i\tilde{x}_ix~i​和x~j\tilde{x}_jx~j​经过同一个编码器f(⋅)f(\cdot)f(⋅)得到特征hih_ihi​和hjh_jhj​, 然后再经过一个投影层(Linear+ReLU+Linear), 得到ziz_izi​和zjz_jzj​

加了一个全连接层, 直接在ImageNet上面涨点10个点
使用的损失函数如下, 一个样本有1个正样本, 和2∗N−12*N-12∗N−1个负样本

ℓi,j=−log⁡exp⁡(sin⁡(zi,zj)/τ)∑k=12N1[k≠i]exp⁡(sin⁡(zi,zk)/τ)\ell_{i,j}=-\log\frac{\exp(\sin(\boldsymbol{z}_i,\boldsymbol{z}_j)/\tau)}{\sum_{k=1}^{2N}\mathbb{1}_{[k\neq i]}\exp(\sin(\boldsymbol{z}_i,\boldsymbol{z}_k)/\tau)} ℓi,j​=−log∑k=12N​1[k​=i]​exp(sin(zi​,zk​)/τ)exp(sin(zi​,zj​)/τ)​

需要注意的一点是投影的g(⋅)g(\cdot)g(⋅)只在训练时使用, 在做下游任务的时候没有这个g(⋅)g(\cdot)g(⋅)

SimCLR和Inva Spread的区别:

  • 使用了更多的数据增强
  • 加了一个投影的g(⋅)g(\cdot)g(⋅)函数
  • 使用了更大的batch size(8192), 训练的时间更久

方法

更多的数据增强



作者进行了两两数据增强的融合, 最后发现随机的裁剪和随机的色彩变换效果最好

加一个全连接层


Linear表示去掉全连接层后面的ReLU激活函数
Non-linear表示全连接层加ReLU激活函数
None表示去掉全连接层和ReLU激活函数
横坐标表示投影后的特征维度
观察到两个现象:

  • 不加全连接和加全连接差距非常明显
  • 投影后的特征维度大小对结果影响不大

MoCov2: Improved Baselines with Momentum Contrastive Learning

论文介绍

参考了SimCLR, 使用了全连接层作为projection head,以及使用了更多的数据增强来提升baseline
总共就四个改动

  • 加了一个MLP层
  • 使用了更多的数据增强
  • 学习率使用了cosine learning rate schedule
  • epoch从200变成了800

无监督学习模型越大, 训练越久效果就会越好, MoCo v2使用更少的epoch, 效果比SimCLR效果好

MoCo v2比其他端到端的方法更加节省内存

SimCLR v2: Big Self-Supervised Models are Strong Semi-Supervised Learners

论文介绍


文章做了三部分:

  1. 第一部分将SimCLR v1变成 SimCLR v2
  2. 第二部分使用有监督的微调
  3. 第三部分用模型生成伪标签然后一起训练

这里只讲第一部分, 做了以下的改进

  • 使用了更大的模型(152层的ResNet), 通道数是原来的3倍, 使用了selective kernel(见下图)
  • 将SimCLR v1中的FC+ReLU变成FC+ReLU+FC+ReLU, 加深了projection head
  • 也使用了动量编码器

SwAV: Unsupervised Learning of Visual Features by Contrasting Cluster Assignments

论文介绍

将对比学习和聚类的方法结合在一起

左边就是之前对比学习用的方法, 右边是本文的方法
作者不想和那么多负样本比, 而是想和聚类中心比
聚类中心就是右图的CCC, 尺寸为D×KD\times KD×K, 其中DDD是特征维度, KKK是聚类中心的个数, 文中取的是3000, Z1Z_1Z1​和Z2Z_2Z2​的尺寸是B×DB\times DB×D
作者的想法是如果两个Z1Z_1Z1​和Z2Z_2Z2​的特征足够相似, 那么给定聚类中心以及Z_1, 他们进行点乘得到Q1Q_1Q1​, 聚类中心以及Z2Z_2Z2​点乘得到Q2Q_2Q2​, 那么应该可以用Q1Q_1Q1​来预测Q2Q_2Q2​, 且用Q2Q_2Q2​来预测Q1Q_1Q1​

为什么用聚类中心

  1. 如果和所有的类做对比, 那么负样本太多
  2. 如果选择一部分, 那么不具有代表性

因此作者想用有代表性的一部分来表示整个集合

SwAV的效果是最好的

方法

Multi-crop

之前的方法都是做两个数据增强(crop), 也就是裁剪了两张图片, 形成一个正样本对. 本文提出的是多进行一些裁剪, 但是多裁剪会导致计算量的增大
原来是将图像resize成256*256, 然后裁剪出两个224 * 224
本文的方法是选择两个160*160的crop和四个96*96的crop, 这样就会有6个视角, 增加了正样本, 并且模型的计算量没有增加很多

BYOL: Bootstrap Your Own Latent A New Approach to Self-Supervised Learning

论文介绍

没有负样本的约束
在之前的对比学习中, 如果没有负样本的约束, 就会导致模型对于任何样本都输出自身, 因为自身和自身肯定都是正样本, 这就导致模型容易学到一个捷径解
BYOL的模型图如下

首先输入图像xxx经过数据增强ttt和t′t^{'}t′得到不同视角的图片vvv和v′v^{'}v′, 然后分别经过编码器fθf_{\theta}fθ​和fξf_{\xi}fξ​得到特征yθy_{\theta}yθ​和yξ′y_{\xi}^{'}yξ′​, 其中fξf_{\xi}fξ​是由fθf_{\theta}fθ​经过指数移动平均计算得到的. 然后经过两个投射层(FC+ReLU)gθg_{\theta}gθ​和gξg_{\xi}gξ​得到zθz_{\theta}zθ​和zξ′z_{\xi}^{'}zξ′​, 其中gξg_{\xi}gξ​是由gθg_{\theta}gθ​经过指数移动平均计算得到的.
以往的方法是希望zθz_{\theta}zθ​和zξ′z_{\xi}^{'}zξ′​尽可能的一致
但是BYOL加了一个新的层qθq_{\theta}qθ​, 这个qθq_{\theta}qθ​和gθg_{\theta}gθ​的网络结构是一样的, 得到一个新的特征qθ(zθ)q_{\theta}(z_{\theta})qθ​(zθ​), 作者希望它们的预测和zξ′z_{\xi}^{'}zξ′​尽可能的一致, 图中的sgsgsg表示没有梯度回传
最后使用yθy_{\theta}yθ​去做下游任务

这个想法和SwAV很像, 都是将匹配的问题转化为预测的问题, 但是SwAV借助了一个聚类中心, 但是BYOL什么都没有

Loss也很简单, 使用的是MSE Loss

Random表示随机初始化的残差网络
BYOL是正确的BYOL
No Normalization是它们一开始复现的错误的网络
LayerNorm也是什么都没学到
作者认为在做BN的时候是存在信息泄漏的, 信息泄漏导致模型看到了其他的样本, 这些样本就相当于是隐式的负样本, 对比学习的隐式任务就是这个图片和归一化的平均图片有什么差别, 但是这个结论在后面被推翻了

BYOL works even without batch statistics

论文介绍

作者不希望让大家认为是BN提供了隐式的负样本, 想要跳出负样本的范畴
作者分别对encoder, Projector, Predictor中是否用BN做了消融实验

但是当Projector使用了BN的时候训练还是失败了
当Encoder和Projector中没有用BN层的时候SimCLR也失败了
这就证明了BYOL成功不是因为BN提供了隐式的负样本
最后得出的结论是BN层能够更加稳定的训练
作者给出的结论是, 如果模型初始化的足够好, 那么模型即使没有BN也能很好的训练
作者使用了Group Norm(归一化)和Weight Standarization(初始化)

作者将骨干网络从ResNet换成了ViT, 当batch size比较小的时候, 曲线比较平滑, 当batch size比较大的时候, 准确度会波动
当每次loss大幅度下降的时候, 梯度也会有一个波峰, 波峰发生在第一层
第一层是patch projection的时候, 就是一个可以训练的全连接层
解决办法很简单, 就是随机初始化patch projection层然后将这一层冻住, 问题就解决了
