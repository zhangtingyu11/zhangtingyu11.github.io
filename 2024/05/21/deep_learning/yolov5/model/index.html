配置文件

配置文件如下图

官方提供5每个目标检测版本: yolov5n, yolov5s, yolov5m, yolov5l, yolov5x
版本之间的网络结果是类似的, 唯一的区别是depth_multiple和width_multiple
其中每一层可能需要重复多次, depth_multiple是每一层重复的倍率
width_multiple是输出通道数的倍率

parse_model函数

parse_model是根据配置文件来解析模型, 代码如下


下面以yolov5n为例, 讲解一下parse_model函数
首先输入参数有两个:

  • d: 配置字典, 包含了一些基础配置
  • ch: 在yolov5n中为[3], 表示输入的通道数, 后续还要存储每个模块的输出通道数

介绍上面几个变量:

  • anchors: 配置文件中的锚框, yolov5n中的配置是[[10, 13, 16, 30, 33, 23], [30, 61, 62, 45, 59, 119], [116, 90, 156, 198, 373, 326]]
  • nc: 种类个数, 因为用的是coco所以种类数是80
  • gd: 深度的倍率(每个模块相比于基础配置重复多少次), yolov5n中设置为0.33
  • gw: 通道数的倍率, yolov5n中设置为0.25
  • act: 应该是激活函数, 设置为None, 也就是用的默认的SiLU
  • ch_mul: 设置为None

    下面的代码是在指定act的情况下, 配置Conv模块的激活函数, 因为设置的是None, 所以在这个没有设置


如果ch_mul为None, 那么就将其设置为8


na表示anchor的数量, no表示输出的维度, 其中no = na * (nc + 5)表示每个anchor有80个输出表示类别, 还有5个输出分别表示x, y, w, h和objectness


创建layers用来存储网络层, 创建save用来保存中间用到的层的编号(因为并不是所有的层的输入都是上一层的输出, 可能存在一些跨层连接), c2用来存储每一层的输出通道数(顺序存储)


遍历backbone和检测头的所有层

具体说一下每一层包含什么, 以conv类为例
[-1, 1, ‘Conv’, [64, 6, 2, 2]]

  • -1: 这个表示哪一层的out_channel表示这一层的in_channel, -1表示上一层的输出通道数是这一层的输入通道数
  • 1: 表示这个模块重复多少次
  • ‘Conv’: 这个表示使用什么模块
  • [64, 6, 2, 2]: 分别表示输出通道数, 卷积核大小, 步长, padding


以c3类为例
[-1, 3, ‘C3’, [128]]

  • -1: 这个表示哪一层的out_channel表示这一层的in_channel, -1表示上一层的输出通道数是这一层的输入通道数
  • 3: 表示这个模块重复多少次
  • ‘C3’: 表示使用什么模块

还有一种配置
[-1, 3, C3, [1024, False]]
其中False表示不使用残差连接


以sppf类为例
[-1, 1, ‘SPPF’, [1024, 5]]

  • -1: 这个表示哪一层的out_channel表示这一层的in_channel, -1表示上一层的输出通道数是这一层的输入通道数
  • 1: 表示这个模块重复多少次
  • ‘SPPF’: 表示使用什么模块
  • [1024, 5]: 分别表示输出通道数和kernel_size


以nn.Upsample类为例
[-1, 1, nn.Upsample, [None, 2, “nearest”]]

  • -1: 这个表示哪一层的out_channel表示这一层的in_channel, -1表示上一层的输出通道数是这一层的输入通道数
  • 1: 表示这个模块重复多少次
  • nn.Upsample: 表示使用什么模块
    -[None, 2, “nearest”]: 分别表示输出尺寸, 缩放的比例, 使用的插值方法. 左边的配置表示放大两倍, 使用最近邻插值. 第一个和第二个参数只能配置其中一个


以concat类为例
[[-1, 4], 1, Concat, [1]]

  • [-1, 4]: 表示对哪几层的输出进行拼接
  • 1: 这个模块重复多少次
  • Concat: 表示使用什么模块
  • [1]: 在什么维度进行拼接


以detect类为例
[[17, 20, 23], 1, Detect, [nc, anchors]]

  • [17, 20, 23]: 获取输出的层有哪些
  • 1: 模块重复的次数
  • Detect: 表示使用什么模块
  • [nc, anchors]: 分别表示类别数和anchor的配置

这边应该是每一层对应一个anchor的配置

Detect类

__init__函数

stride = None表示步长在构建网络的时候动态计算出, 参考后续更改
dynamic = False表示要强制构建anchor网格
export=False 表示如果是预测阶段, 就需要输出最终的结果和构建anchor的特征x


__init__函数有几个输入

  • nc: 类别的数量
  • anchors: 锚框的配置
  • ch: 每一个锚框的配置所使用的输入通道数
  • inplace: 是否是原地操作

类的属性有一下几个

  • self.nc: 类别的数量
  • self.no: 类别的数量+5, 表示每个锚框需要输出类别, x, y, w, h, objectness
  • self.nl: 生成anchor的层数
  • self.na: 每一层anchor有几种基础配置, 因为只设置anchor的高宽, 索引用len(anchors[0]) // 2计算
  • self.grid: 初始化每一层的网格是空的, 后续会记录特征图上每个anchor的坐标
  • self.anchor_grid: 初始化每一层的锚框网格是空的, 后续会记录特征图上每个anchor的尺寸
  • self.anchors: 一个尺寸为(3, 3, 2)的张量, 用register_buffer进行注册, 用这个注册的变量不会被优化器更新
  • self.m: 长度为3的nn.ModuleList, 每个元素都是个2D卷积, 用于处理对应的anchor层
  • self.inplace: 是否是原地操作
