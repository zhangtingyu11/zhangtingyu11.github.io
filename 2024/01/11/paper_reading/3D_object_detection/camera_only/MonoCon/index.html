Learning Auxiliary Monocular Contexts Helps Monocular 3D Object Detection

发表于2024-01-11|更新于2024-01-11|论文阅读3D目标检测纯相机3D目标检测
|字数总计:2.4k|阅读时长:10分钟|阅读量:

主要思路

在模型中添加一些辅助的检测头来提升模型的性能

Introduction

单目3D目标检测的主要挑战之一是精确的预测物体的3D中心位置

目前SOTA的方法主要有两类, 可以根据是够需要额外的信息(LiDAR的深度, 单目相机的深度)来进行分类

在实际应用中, 3D中心点的位置可以分解为3D中心点在图像上的投影点和物体的深度. 再结合相机的内参, 就可以得到3D中心点的位置

本文主要是关注端到端的单目3D目标检测, 并且不利用任何的额外信息

在实现上, MonoCon使用一个很简单的设计, 包含三部分: 一个DNN, 一系列的检测头用于3D目标检测, 一系列的检测头用于学习辅助的语义信息

主要贡献

  • 提出了一个简单但是有效的方法, MonoCon, 通过学习辅助的单目context来辅助进行单目3D目标检测
  • SOTA
  • 激发了探索更多辅助的任务

Approach

Problem Definition

给定图像网格ΛΛΛ (在KITTI上是384×1280384 \times 1280384×1280), IΛI_ΛIΛ​是图像网格上的图像. 3D包围框可以表示为物体的3D中心点位置(x,y,z)(x, y, z)(x,y,z), 物体的尺寸信息(h,w,l)(h, w, l)(h,w,l)和物体的观测角α\alphaα

Challenges

mAP在中心点位置预测相对准确的情况下对于尺寸和航向并不敏感
但是即使尺寸和航向预测的很准确, 只要3D中心点预测的不准, 那么就会导致mAP很低, 因为尺寸的偏差也就1~3m, 但是3D中心点的偏差有1~60m

有两种不同的方法来预测投影的3D中心点(xc,yc)(x_c, y_c)(xc​,yc​)
一种是直接通过学习一个heatmap来预测, 在图像外的中心点要么直接丢弃(SMOKE采用的策略) 或者使用2D中心点和投影的3D中心点的连线与图像边缘的角点(MonoFlex的方法)
另一种方法是将投影的3D中心点表示为2D包围框的中心点(xb,yb)(x_b, y_b)(xb​,yb​)和偏差(Δx,Δy)(\Delta x, \Delta y)(Δx,Δy). 这样投影的3D中心点就表示为xc=xb+Δx,yc=yb+Δyx_c = x_b + \Delta x, y_c = y_b + \Delta yxc​=xb​+Δx,yc​=yb​+Δy
由于offset的偏差很大, 第二种方法很难学习
本文提出, 只要有足够的单目相机的语义信息, 那么后面一种方法也可以学习的很好

The Proposed MonoCon Method

Feature Backbone

给定一张输入图像IΛI_ΛIΛ​, 尺寸为3×H×W3 \times H \times W3×H×W
backbone用来求输出特征图FFF, 尺寸为D×h×wD \times h \times wD×h×w

F=f(IΛ;Θ)F=f(I_\Lambda;\Theta) F=f(IΛ​;Θ)

其中Θ\ThetaΘ是所有的可学习的参数, DDD是输出的特征图的特征维度, h,wh, wh,w由下采样倍率sss决定
使用DLA-34作为backbone

The 3D Bounding Box Regression Heads

学习和类别相关的2D包围框中心点heatmap Hb\mathcal{H}^{b}Hb, 尺寸为c×h×wc \times h \times wc×h×w, 其中ccc是类别数

Hb=g(F;Φb)\mathcal{H}^b=g(F;\Phi^b) Hb=g(F;Φb)

其中g(⋅;Φb)g(\cdot;\Phi^b)g(⋅;Φb)是一个轻量化的模块, 有着可学习的参数Φb\Phi^bΦb

F→Conv+AN+ReLUFd×h×w→ConvHc×h×wb\begin{aligned}F\xrightarrow{Conv+AN+ReLU}\mathbb{F}_{d\times h\times w}\xrightarrow{Conv}\mathbb{H}_{c\times h\times w}^b\end{aligned} FConv+AN+ReLU​Fd×h×w​Conv​Hc×h×wb​​

第一个卷积用于缩减特征维度, ANANAN表示Attentive Normalization

用于计算2D包围框的中心点到投影的3D中心点的offset可以定义为

O2×h×wc=g(F;Θbc)\mathcal{O}_{2\times h\times w}^c=g(F;\Theta^{b_c}) O2×h×wc​=g(F;Θbc​)

相似地, 深度和尺寸可以用以下的方法回归

Z1×h×w=1Sigmoid(g(F;ΘZ)[0])+ϵ−1,σ1×h×wZ=g(F;ΘZ)[1],S3×h×w3D=g(F;ΘS3D),\begin{aligned} &\mathcal{Z}_{1\times h\times w} =\frac1{\text{Sigmoid}\left(g\left(F;\Theta^{Z}\right)[0]\right)+\epsilon}-1, \\ &\sigma_{1\times h\times w}^{\mathcal{Z}} =g(F;\Theta^Z)[1], \\ &\mathcal{S}_{3\times h\times w}^{3D} =g(F;\Theta^{S^{3D}}), \end{aligned} ​Z1×h×w​=Sigmoid(g(F;ΘZ)[0])+ϵ1​−1,σ1×h×wZ​=g(F;ΘZ)[1],S3×h×w3D​=g(F;ΘS3D),​

其中g(F;ΘZ)g(F;\Theta^Z)g(F;ΘZ)预测深度和它的不确定性

对于观测角α\alphaα, 使用multi-bin的配置, 将角度分成bbb个不重叠的bin, 观测角的检测头定义为

A2b×h×w=g(F;ΘA)\begin{aligned}\mathcal{A}_{2b\times h\times w}=g(F;\Theta^A)\end{aligned} A2b×h×w​=g(F;ΘA)​

αi\alpha_iαi​是前bbb个通道, 通过softmax求出属于每个bin的概率, αr\alpha_rαr​是后bbb个通道, 表示在这个bin内部的angle residual

Computing the predicted 3D bounding box

基于Hb\mathcal{H}^bHb的每个通道的极值点, 通过NMS以及阈值τ\tauτ进行过滤, 对于每个类别都会检测一系列的2D包围框的中心点
通过获得heatmap的2D包围框中心点坐标和预测的offset得到投影的3D中心点坐标
最后通过预测的包围框和观测角得到最终的3D包围框

The Auxiliary Context Regression Heads

(1) 投影的关键点的heatmap. 预测8个角点和投影的3D中心点的heatmap

H9×h×wk=g(F;Θk)\mathcal{H}_{9\times h\times w}^{k}=g(F;\Theta^{k}) H9×h×wk​=g(F;Θk)

(2)预测和8点投影的角点offset

O16×h×wk=g(F;Θbk)\mathcal{O}_{16\times h\times w}^k=g(F;\Theta^{b_k}) O16×h×wk​=g(F;Θbk​)

这里的ggg和预测2D中心点到3D中心点的offset在第一个卷积块上的参数是共享的

(3)2D包围框的包围框尺寸

S2×h×w2D=g(F;ΘS2D)\mathcal{S}_{2\times h\times w}^{2D}=g(F;\Theta^{S^{2D}}) S2×h×w2D​=g(F;ΘS2D)

(4)关键点位置因为下采样产生的residual
假设车辆的2D包围框中心点为(xb∗,yb∗)(x_b^*, y_b^*)(xb∗​,yb∗​), 因为下采样, 它在特征图中的位置为xb=⌊xb∗s⌋,yb=⌊yb∗s⌋x_{b}=\lfloor\frac{x_{b}^{*}}{s}\rfloor,y_{b}=\lfloor\frac{y_{b}^{*}}{s}\rfloorxb​=⌊sxb∗​​⌋,yb​=⌊syb∗​​⌋
那么残差可以通过下面的公式算出

分别计算2D包围框中心点和9个关键点的残差(对每个关键点都求一次, 相当于下面第二个公式重复9次)

R2×h×wb=g(F;ΘRb)R2×h×wk=g(F;ΘRk)\begin{aligned}\mathcal{R}_{2\times h\times w}^b&=g(F;\Theta^{R^b})\\\mathcal{R}_{2\times h\times w}^k&=g(F;\Theta^{R^k})\end{aligned} R2×h×wb​R2×h×wk​​=g(F;ΘRb)=g(F;ΘRk)​

Loss Functions

损失函数包含以下部分:

  • Gaussian kernel weighted focal loss 用于计算heatmap的损失
  • Laplacian aleatoric uncertainty loss function (GUPNet中的loss)用于深度估计
  • dimension-aware L1 loss用于计算尺寸
  • 交叉熵损失函数用于计算角度的bin index
  • 标准的L1 loss用于计算offset, 角度的残差, 因为下采样导致的残差

(1) The Gaussian kernel weighted focal loss function
假设H1×h×w\mathcal{H}_{1\times h \times w}H1×h×w​是预测的heatmap, gt heatmap是H1×h×w∗\mathcal{H}^*_{1\times h \times w}H1×h×w∗​
对于每个gt中心点(xb∗,yb∗)(x_b^*, y_b^*)(xb∗​,yb∗​), 那么在当前特征图下的gt中心点为xb=⌊xb∗s⌋,yb=⌊yb∗s⌋x_{b}=\lfloor\frac{x_{b}^{*}}{s}\rfloor,y_{b}=\lfloor\frac{y_{b}^{*}}{s}\rfloorxb​=⌊sxb∗​​⌋,yb​=⌊syb∗​​⌋, 然后根据高斯核G(x,y)=exp⁡(−(x−xb)2+(y−yb)22⋅σb2)G(x,y) = \exp{(-\frac{(x-x_{b})^{2}+(y-y_{b})^{2}}{2\cdot\sigma_{b}^{2}})}G(x,y)=exp(−2⋅σb2​(x−xb​)2+(y−yb​)2​)来构建热力图, 如果两个高斯核重叠, 那么取最大的元素值

(2) The Laplacian aleatoric uncertainty loss function for depth
创建一个稀疏的gt深度图Z1×h×w∗\mathcal{Z}_{1\times h\times w}^{*}Z1×h×w∗​, 一个物体的gt深度会对应到它的2D包围框的中心点

将深度附给2D包围框中心点比较巧妙, 之前的方法都是赋值给3D投影点, 但是3D投影点会在图像外
